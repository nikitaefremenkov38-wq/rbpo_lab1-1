# Amusement Park Service

Тема проекта: сервис управления парком аттракционов.

## Основные сущности
- `Attraction` — аттракцион (название, описание, вместимость).
- `Schedule` — слот работы аттракциона.
- `Maintenance` — окно технического обслуживания аттракциона.
- `Visitor` — посетитель парка.
- `Ticket` — билет посетителя на конкретный слот.

## Что умеет сервис
- CRUD для всех сущностей (`attractions`, `schedules`, `maintenances`, `visitors`, `tickets`) через PostgreSQL.
- Бизнес-операции:
  - массовая покупка билетов для посетителя;
  - перенос билета на другой слот;
  - планирование обслуживания с проверкой конфликтов по проданным билетам;
  - деактивация посетителя с отменой будущих билетов;
  - получение маршрута посетителя за период.

## База данных и конфигурация
- Используется PostgreSQL.
- Чувствительные параметры БД вынесены в переменные окружения (`.env`, пример в `.env.example`).
- Таблицы создаются JPA/Hibernate, тестовые данные загружаются из `src/main/resources/data.sql`.
